## Visao Geral

Site institucional desenvolvido para a **GWR Engenharia Eletrica**, empresa especializada em projetos eletricos, subestacoes, SPDA e laudos tecnicos em Minas Gerais.

Em producao em **AWS Lightsail** com deploy containerizado via Docker + nginx.

## Requisitos do Cliente

- Presenca digital profissional orientada a captacao de leads em todo o estado de MG.
- Ranqueamento local por cidade para os servicos mais buscados (SPDA, projetos eletricos, subestacoes).
- Landing page de alta conversao para campanhas de trafego pago, sem interferir no SEO organico.
- Velocidade e responsividade no mobile (maioria do publico-alvo).
- Custo de hospedagem baixo e previsivel.

## Arquitetura e Stack

### Framework — Next.js 16 + SSG

- **App Router** com Static Site Generation para maximo desempenho e indexacao.
- **TypeScript** em toda a base de codigo.
- **Tailwind CSS 4** para estilizacao responsiva mobile-first.
- **Framer Motion 12** para animacoes de entrada e transicoes de secao.
- **React Hook Form + Zod** para validacao de formularios de orcamento.

### SEO Tecnico

- **Metadata** completa por rota: `title`, `description`, `canonical`, Open Graph e Twitter Cards.
- **Structured Data JSON-LD** para `LocalBusiness` (NAP, area de atuacao) e `FAQPage` em paginas de servico.
- **Sitemap** automatico em `/sitemap.xml` e `robots.txt` configurado.
- **Imagens** otimizadas com Next.js Image (avif/webp, cache de 30 dias).
- **13+ cidades de MG** com paginas dedicadas `/cidades/[slug]` para SEO local sem canibalizacao.

### Landing Page para Trafego Pago

- Rota dedicada com `robots: noindex` para nao competir com paginas organicas.
- Sem navegacao principal — foco total na conversao.
- CTA direto para WhatsApp e formulario de orcamento simplificado.

### Paginas de Servico

| Servico | Conteudo |
|---|---|
| SPDA | Hub completo: projeto, instalacao, laudo (NBR 5419 + ART) |
| Projetos Eletricos | Residencial, comercial e industrial (NBR 5410) |
| Subestacoes | Aereas e abrigadas, AT/MT |
| Eficiencia Energetica | Diagnostico e adequacao |
| Laudos Tecnicos | NR-10, SPDA, instalacoes existentes |

### Infraestrutura — Docker + AWS

Pipeline de deploy containerizado:

```
Build local → Docker image → AWS ECR → AWS Lightsail → nginx (SSL + cache)
```

- **Dockerfile** multi-stage para imagens enxutas em producao.
- **nginx** como reverse proxy com HTTPS e cache de assets estaticos.
- **Script de deploy** com suporte a rollback para versao anterior.
- **docker-compose** no servidor para orquestracao dos containers.

## Decisoes Tecnicas

- **SSG em vez de SSR**: Conteudo predominantemente estatico — SSG entrega performance maxima e elimina cold starts.
- **AWS Lightsail em vez de Vercel**: Controle total do servidor, SSL sem limites de dominio e custo mensal fixo previsivel.
- **Docker em vez de deploy direto**: Portabilidade total — o mesmo container pode migrar para EC2 ou ECS sem reconfigurar o ambiente.
- **Landing pages com `noindex` separadas**: Isola trafego pago do organico, protegendo o ranqueamento das paginas principais.
