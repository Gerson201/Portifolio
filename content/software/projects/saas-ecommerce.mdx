## Visão Geral

Plataforma de e-commerce **Software as a Service** que permite a múltiplos lojistas criarem e gerenciarem suas lojas online de forma independente. Cada tenant possui isolamento completo de dados, customizacao visual via editor drag-and-drop e integração com múltiplos gateways de pagamento.

O projeto utiliza arquitetura **monorepo com Turborepo e pnpm**, separando backend, frontend e pacotes compartilhados com build otimizado e cache inteligente.

## Arquitetura

### Backend — NestJS
- API RESTful com **NestJS** (porta 3333), seguindo arquitetura modular.
- ORM **Prisma** com PostgreSQL para persistencia de dados multi-tenant.
- Autenticacao **JWT** com refresh tokens e sessoes seguras.
- **RBAC** (Role-Based Access Control) com 3 níveis: Admin, Gestor e Operador.
- Middleware de isolamento de tenant por subdominio ou header.
- Documentacao de API auto-gerada com Swagger/OpenAPI.

### Frontend — Next.js 14
- Aplicação **Next.js 14** (porta 3000) com App Router e Server Components.
- Gerenciamento de estado global com **Zustand** (leve e performatico).
- Renderizacao hibrida: SSR para SEO, CSR para interatividade.
- Design responsivo e acessível.

### Monorepo
- Gerenciado com **Turborepo** para builds incrementais e cache distribuido.
- **pnpm** workspaces para gerenciamento eficiente de dependencias.
- Pacotes compartilhados (types, utils, configs) reutilizados entre apps.

## Editor Visual

- Interface **drag-and-drop** construida com `@dnd-kit` para montagem de paginas.
- Componentes pre-construidos: Hero, Grid de Produtos, Banner, Carousel, Footer, etc.
- Preview em tempo real das alteracoes.
- Salvamento automático e versionamento de layouts.
- Sem necessidade de conhecimento técnico por parte do lojista.

## Gateways de Pagamento

Integração com **4 gateways** para máxima flexibilidade:

- **Pagar.me**: Cartao de credito, boleto e Pix.
- **Stripe**: Pagamentos internacionais com cartao.
- **Mercado Pago**: Cartao, boleto, Pix e checkout transparente.
- **Asaas**: Cobranca recorrente, boleto e Pix com split de pagamento.

Cada tenant pode configurar seu próprio gateway preferido com chaves independentes.

## Multi-Tenancy

- Isolamento completo de dados por tenant no PostgreSQL via Prisma.
- Customizacao de tema, logo, dominio e configurações por loja.

- Painel administrativo master para gestao de todos os tenants.
