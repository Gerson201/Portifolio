## Visao Geral

Solucao digital completa desenvolvida para a **RGA Engenharia** — empresa especializada em automacao industrial, engenharia eletrica e mecanica — que combina presenca institucional publica com plataforma interna de gestao empresarial integrada ao ERP Bling.

O sistema esta em producao em **AWS Lightsail** com 725 produtos WEG sincronizados e suporte a e-commerce B2B.

## Arquitetura

### Backend — Django 5.2 + DRF

- **Autenticacao**: Token DRF + sessao Django com registro por verificacao de e-mail e reenvio.
- **Seguranca**: Rate limiting por escopo (auth, CEP, operacoes criticas), CORS, CSP headers e validador de senha forte.
- **Criptografia de PII**: Campos sensiveis (CPF, CNPJ, e-mail, telefone) criptografados com Fernet AES-128-CBC + HMAC-SHA256 via `django-fernet-fields` — vazamento de backup nao expoe dados de clientes.
- **Multi-banco**: Separacao de contextos com SQLite/PostgreSQL por dominio de negocio — cada area opera com seu proprio banco de dados.
- **Proxy CEP**: Endpoint interno que consulta ViaCEP para evitar CORS no frontend, com throttle configurado.

### Frontend — Angular 20+

- Interceptor HTTP global para autenticacao automatica em todas as requisicoes protegidas.
- `AuthSessionService` com persistencia segura em `localStorage`.
- Notificacoes em tempo real (toasts) para feedback de operacoes.
- Utilitario proprio para parsing de erros DRF aninhados.

### Modulos do Sistema

O backend esta organizado em sete modulos independentes:

| Modulo | Responsabilidade |
|---|---|
| `authentication` | Identidade, registro, login, verificacao de e-mail, controle de acesso por roles |
| `core` | Infraestrutura compartilhada, proxy CEP, utilitarios de validacao |
| `ecommerce` | Catalogo, carrinho, checkout, pedidos, area do cliente |
| `integrations` | Bling OAuth 2.0, sincronizacao WEG, Asaas (pagamentos) |
| `management` | Operacoes internas e controle de estoque |
| `crm` | Gestao de relacionamento e historico de clientes |
| `blog` | Conteudo tecnico e SEO |

### Integracao ERP — Bling

- **OAuth 2.0** completo com renovacao automatica de tokens.
- **Sincronizacao WEG**: 725 produtos importados com precos calculados (margem 35% + custo operacional 15%).
- **Notas Fiscais**: Emissao automatica integrada.
- **Cache e Retry**: Reducao de chamadas a API com backoff exponencial via `tenacity`.

### E-commerce B2B

Fluxo de checkout em tres etapas: cotacao → verificacao OTP → confirmacao de pagamento.

- Carrinho com snapshot de abandono no servidor para recuperacao.
- Token OTP baseado em HMAC com limite de tentativas para confirmacao de pedidos.
- PII mascarada nas respostas da API; sem armazenamento de dados de cartao.
- Integracao com **Asaas** (PIX, boleto PJ, cartao) — boleto restrito a CNPJ valido.
- Area do cliente para historico de pedidos e carrinho salvo com guarda de rota dedicada.

### Infraestrutura AWS

- **AWS Lightsail** para servidor de producao com custo controlado.
- **AWS Cognito** para autenticacao federada e gerenciamento de identidade.
- Preparado para expansao com **Lambda** (serverless) e **S3** (armazenamento de arquivos).

## Site Institucional

A presenca publica da empresa inclui:

- Paginas de servicos por area (Eletrica, Mecanica, Automacao Industrial, Fotovoltaico).
- Cobertura geografica por regiao de atuacao.
- Formulario de contato e orcamento com envio por e-mail.
- Blog tecnico para autoridade e SEO.

## Decisoes Tecnicas

- **Django + Angular em vez de Next.js full-stack**: Separacao clara entre API e SPA permite evoluir cada camada independentemente e reutilizar o backend para integracoes futuras.
- **Multi-banco por dominio**: Isola contextos de negocio, simplifica backups seletivos e facilita eventuais migracoes de dominios especificos para microservicos.
- **Fernet para PII**: Criptografia no nivel do campo garante que vazamentos de backup nao expoe dados sensiveis de clientes.
- **AWS Lightsail**: Custo previsivel para o volume atual com caminho claro de migracao para EC2/ECS se a demanda crescer.
